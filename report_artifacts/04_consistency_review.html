Codebase Understanding

Dataset: 78,390 unique Spotify tracks (v6 pipeline, cross-ID deduplicated), of which 2,157 appeared on the Billboard Hot 100 between 1958–2024 (2.75% positive rate). Enriched with Spotify API artist features, Last.fm listener counts (log-transformed), and VADER lyric sentiment for a charted-track subset of n = 856 (40.1% match rate).

Four models: Logistic Regression (AUC-ROC = 0.914, PR-AUC = 0.275), XGBoost (AUC-ROC = 0.966, PR-AUC = 0.441), Cox Proportional Hazards (C-stat = 0.753, n = 856, mode-stratified), Log-OLS robustness check (R² = 0.212, n = 856).

15 classification features: 10 Spotify audio metrics + 5 artist-level commercial signals. Longevity models add decade_idx and 4 lyric sentiment features. energy, danceability, and tempo removed for VIF > 10. All final numbers in report appendix tables verified against outputs/ CSV files — odds ratios, Cox coefficients, threshold analysis, and model performance all match exactly.

Must-Fix Before Submission

Critical — appears in Executive Summary and Conclusions

The decade_idx Cox finding is stated with the wrong direction in two of the three highest-visibility sections of the report (Executive Summary and Conclusions), while the Cox table in the same appendix states it correctly. A grader reading the report will encounter directly contradictory claims about the same finding.

The correct interpretation: Cox coefficient = −0.666, HR = 0.514. Because HR < 1, more recent decades are associated with a lower exit hazard — songs stay on the chart longer. This is independently confirmed by the Log-OLS decade_idx coefficient of +0.275 (more log-weeks in recent decades). Both models agree.

Affected text (Executive Summary): "rotate off the chart significantly faster" → should be "remain on the chart longer".
Affected text (Conclusions): "49% increase in per-week exit hazard" → should be "49% lower per-week exit hazard (HR = 0.514)".

Critical Issues

Critical decade_idx direction wrong in Executive Summary Executive Summary, p. 2

The Executive Summary states that songs in more recent decades exit the chart faster. The Cox model shows the opposite.

Report says (wrong)
"tracks from more recent decades rotate off the chart significantly faster on a per-week basis than their predecessors"
Cox output (HR = 0.514 < 1)
HR < 1 = lower exit hazard = songs exit slower = longer chart runs in more recent decades.

The Results section (p. 9, Finding 3) and the Cox table (Appendix, p. 18) both state the correct direction. The Executive Summary contradicts them.

Critical decade_idx hazard direction wrong in Conclusions Conclusions, p. 11 + Appendix Cox table, p. 18

The Conclusions and the Cox coefficient table are on adjacent pages and state directly opposite things about the same number.

Conclusions (wrong)
"each additional decade corresponding to a 49% increase in per-week exit hazard"
Cox table, same appendix (correct)
decade_idx: HR = 0.514 → "49% lower exit hazard per decade"

HR = 0.514 means 1 − 0.514 = 0.486, i.e., ~49% lower hazard, not 49% higher. The Conclusions text inverts the result. The Log-OLS coefficient (+0.275) independently confirms the correct direction (more recent = more log-weeks).

Moderate Issues

Moderate Progressive OLS models in Results were never described in Methods Methods (p. 5) → Results 4.2 (p. 9)

Methods describes four models: LR, XGBoost, Cox PH, and Log-OLS (all on n = 856 for longevity). Results 4.2 leads with a progressive OLS enrichment sequence (audio-only → +genre → +artist → full) run on a different sample (n = 2,157) with a different feature set (includes danceability, energy, tempo, time_signature, genre dummies). These intermediate models are never established in Methods, creating a disconnect between the two sections.

Moderate tempo listed as a classification feature in Results Results 4.1, p. 6
Report says
"audio characteristics (valence, tempo, acousticness, instrumentalness, and others)"

tempo was removed from all final models for VIF = 10.7 (above the VIF > 10 removal threshold stated in the Data section). It is not among the 15 classification features. Listing it here implies it was used.

Moderate n = 2,157 OLS and n = 856 Cox/OLS not clearly distinguished Results 4.2 throughout

Results 4.2 moves between two different longevity analyses without signaling the switch to the reader: (1) progressive OLS on n = 2,157 full charted tracks, which yields R² values of 0.047, 0.27, 0.23, and 0.24; and (2) the primary Cox PH and Log-OLS on n = 856 (lyric-matched), which yields C-stat = 0.753 and R² = 0.212. The model performance table in the appendix reports the n = 856 results. A reader comparing the R² values in the text to the table will find an apparent mismatch (0.24 vs. 0.212) unless the sample difference is explicit.

Moderate artist_popularity_api Cox sign: VIF caveat and substantive interpretation conflict Appendix Cox table (p. 18) + Variable definitions (p. 13)

The variable definitions table (p. 13) says of artist_popularity_api: "Highly collinear with artist_peak_popularity (VIF = 14.1); individual coefficient signs should not be interpreted in isolation."

The Cox coefficient table footnote (p. 18) then offers a substantive interpretation of the sign reversal: "one hypothesis is that historical peak popularity captures durable fanbase depth…while current API popularity may partially reflect short-lived viral attention."

These two framings are in tension. The VIF caveat instructs readers not to interpret the sign; the footnote does exactly that. The VIF caveat should be the dominant framing.

Moderate Data cleaning pipeline table shows 61,871 tracks for Q1, but 78,390 are used Appendix, data cleaning table, p. 12

The table shows "Adding artist features → 61,871 unique tracks with artist features (Q1)", implying the classification uses only those tracks with full artist feature coverage. In reality, all 78,390 tracks are used for classification — artist features with missing values are handled via imputation (is_us_artist NaN→0) and the auto-detection threshold. The Results section confirms this: 80% of 78,390 = 62,712 training tracks.

Minor Issues

Minor Conclusion #4 has no supporting exhibit Conclusions, p. 10
Stated
"4. All factors influencing charting longevity have seen absolute correlation decrease over time"

This specific claim does not correspond to any model output, table, or figure in the report. It may be intended to paraphrase the temporal validation finding (all models produce negative out-of-time R²), but that finding is about model generalizability, not correlation magnitudes. Should be reworded to reference a specific result, or removed.

Minor danceability and tempo cited as key features in Introduction Introduction, p. 3

The Introduction mentions "valence, instrumentalness, danceability, tempo, and others" as the audio features driving the analysis. Both were removed from all final models for VIF > 10 (danceability VIF = 12.4, tempo VIF = 10.7). Citing them in the Introduction as illustrative descriptors is not incorrect per se, but it sets a framing that the Data section then removes without explicit callback.

Minor Cox mode stratification rationale is outdated Methods, p. 5–6

The Methods section explains that mode is stratified because it "violated this assumption" (Schoenfeld test). In the v6 pipeline, the features that actually fail the Schoenfeld test are decade_idx, acousticness, duration_min, and sentiment_pos — not mode. Mode is stratified by design (carried over from prior pipeline iterations). The stated rationale is technically a carryover from an earlier version.

Minor Positive rate expressed three different ways Executive Summary, Figure 1 label, Section 2.2

The same quantity appears as "roughly 3%" (Executive Summary), "2.8%" (Figure 1 bar label), and "2.75%" (Section 2.2, precision-recall baseline, threshold table). Should be standardized to 2.75% throughout, with the figure label corrected to match.

Minor Preliminary OLS appendix models include VIF-removed features without explanation Appendix, Model 1 OLS output, p. 19

The "Model 1: Audio features only" OLS regression output includes danceability, energy, tempo, and time_signature — features that the Data section states were removed for VIF > 10. A reader coming from Section II will be confused to see them appear here. A clarifying note that these preliminary models use the full Spotify feature set prior to VIF screening would resolve the inconsistency.
